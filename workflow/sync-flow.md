# 内容同步工作流

## 流程概述
用户输入源内容 → 识别源平台和目标平台 → 内容提取 → 平台适配转换 → 合规审查 → 输出

## 详细步骤

### Step 1: 输入识别
- 用户提供 `source_content`（源内容）
- 用户指定 `source_platform`（weixin/xiaohongshu/douyin）
- 用户指定 `target_platform`（目标平台）
- 可选：`content_type`、`brand_voice`

### Step 2: 内容提取
- 如果是URL链接，尝试抓取内容
- 如果是纯文本，直接使用
- 提取标题、正文、关键信息
- 对于视频内容，提取口播文案

### Step 3: 源平台特征分析
- 识别源平台的内容风格特征
- 标记需要保留的核心信息
- 识别需要转换的格式元素

### Step 4: 平台适配转换
根据源平台→目标平台的路径调用对应prompt：

| 源 | 目标 | 调用prompt |
|----|------|-----------|
| 公众号 | 小红书 | weixin-to-xiaohongshu |
| 小红书 | 抖音 | xiaohongshu-to-douyin |
| 抖音 | 公众号 | douyin-to-weixin |
| 其他组合 | - | 智能适配 |

### Step 5: 合规审查
- 检查敏感词
- 确保不违反目标平台规则
- 检查是否包含引流信息
- 标记需要注意的内容

### Step 6: 输出
- 返回转换后的完整内容
- 附带排版建议
- 提供话题标签建议
- 给出手动微调建议
